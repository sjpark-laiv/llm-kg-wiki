# Wiki: 그래프 탐색 질의 (`/wiki-query`)

사용자가 **`/wiki-query` 뒤에 붙인 질문**에 답한다. 답은 **저장소에 이미 있는 `wiki/` 지식**만으로 구성하며, **`wiki/entity-list.md`를 허브**로 두고 **`wiki/entity/*.md`의 `## 관련 인과관계`(및 동등 절)** → **`wiki/rel/*.md`** 링크를 **따라가며** 근거를 수집한다.

## 호출 형식

- 사용자 입력 예: **`/wiki-query ROE 측면에서 한국 증시 전망을 요약해줘`**
- 명령 본문에는 질문 문자열이 **그대로** 이어진다고 가정한다(앞뒤 공백만 제거).

## 전제

1. 지식의 **진실 공급원은 `wiki/` 마크다운**이다. **없는 사실을 보간하지 않는다.**
2. **허브:** 항상 먼저 **`wiki/entity-list.md`** 를 연다. 여기서 **엔티티 표**, **`rel/` 인덱스 표**, **주제 통합 문서** 섹션을 본다.
3. 스킬 **`.cursor/skills/wiki-knowledge-ingest/SKILL.md`** 의 상대경로·용어 규칙을 따른다(질의는 **읽기 전용**이므로 새 페이지 작성 절차는 **적용하지 않음**).
4. **외부 URL·실시간 시세**는 사용자가 요청하지 않는 한 쓰지 않는다.

## 수행 순서

1. **`wiki/entity-list.md` 전체**를 읽는다.
2. 질문에서 **키워드·고유명사·지표(ROE, PER, 코스피 등)** 를 뽑아 `entity-list` 표의 **표시명·파일 경로**와 매칭한다. 매칭이 애매하면 `rg`/목록으로 `wiki/entity/`, `wiki/rel/`, `wiki/*.md` 를 **보조 검색**한다.
3. **시작 노드**를 고른다 (우선순위):
   - 표에 **직접 해당하는 `entity/<slug>.md`** 가 있으면 그 파일을 연다.
   - 없으면 **`rel/` 인덱스 표** 또는 **`wiki/` 루트 주제 통합 문서**(`entity-list` 하단)에서 가장 가까운 문서를 연다.
4. 열린 각 **`wiki/entity/<slug>.md`** 에서 **`## 관련 인과관계`**(또는 **`## 관련 인과관계 (교차 링크)`** 등 **동일 역할 절**)의 **`../rel/*.md`** 링크를 **큐**에 넣고 순서대로 읽는다.
5. 각 **`wiki/rel/<slug>.md`** 에서:
   - **서두·Entity 정의 표**의 **`../entity/*.md`** 링크를 따라 **엔티티를 역방향 확장**하고,
   - **`#cause` / `#effect` / `#mechanism` / `#evidence` / `#backlinks`** 등 **앵커가 있으면** 질문과 관련 있는 절을 우선 읽는다.
   - **`## 8. 역링크`**·**하위 인과**에 나온 다른 `rel`·`wiki/` 루트 문서를 필요 시 **추가 탐색**한다.
6. **탐색 깊이·너비 제한(권장):** 같은 세션에서 **엔티티 ≤ 12개**, **`rel` ≤ 10개**를 넘기기 전에 멈추고, 부족하면 **“위키에 없음/미수집”** 이라고 적는다. 질문이 좁으면 더 적게 읽어도 된다.
7. **답변 작성:**
   - **요지 → 근거(위키 문장 수준) → 인과 연결** 순으로 정리한다.
   - 근거마다 **`wiki/...` 경로**를 괄호나 불릿으로 **명시**한다(에이전트가 읽은 파일만).
   - **서로 다른 화자·요약**이 섞이면 **출처 구분**(예: 강관우 vs 김장열)을 한 줄이라도 넣는다.
   - **`entity-list`에 없는 주장**은 하지 않는다. 추정은 **“위키에 명시되지 않음”** 으로 표시.

## 산출물

- **질문에 대한 답변 본문**(한국어 기본).
- **탐색 경로 요약:** 읽은 `entity-list` → `entity/` → `rel/` (및 주제 문서) 목록을 짧게 bullet 로 덧붙인다(감사용).

## 금지

- **`wiki/` 밖**을 근거로 한 단정.
- 질의만으로 **`wiki/` 파일을 새로 만들거나 수정**하는 것(사용자가 **저장·갱신**을 명시한 경우는 예외).
- **`entity-list`를 읽지 않고** 바로 `rg`만으로 답을 끝내는 것(단, `entity-list`에 없는 신규 파일만 찾을 때는 보조로 허용).

## 참고

- 엔티티 절 이름이 **`## 관련 인과관계`** 가 아닐 수 있으므로, **`rel/`로 가는 `../rel/` 링크가 있는 절**을 동일 취급한다.
- `entity-list`의 **`wiki/` 루트 주제 통합 문서**는 **여러 `rel`을 한 번에 묶은 요약**이므로, 질문이 넓을 때 **먼저 읽고** 그래프 탐색 범위를 줄인다.
