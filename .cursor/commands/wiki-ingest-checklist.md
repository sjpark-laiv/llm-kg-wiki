# Wiki: 저장 전 체크리스트만 출력

새 정보를 저장하기 **직전**에, 스킬의 체크리스트를 기준으로 **할 일 목록**과 **누락 위험**을 짧게 출력한다. 파일을 쓰지 않고 계획만 세울 때 사용한다.

## 전제

`.cursor/skills/wiki-knowledge-ingest/SKILL.md`의 **작업 체크리스트**, **Named entity** 표, **인과관계 구조화(RCA·Impact)** 절을 기준으로 한다. 입력이 **엔티티·인과 분해**에 적합하면 아래 **그래프 확장 체크리스트**도 같이 적용한다.

## 입력

- 사용자가 붙여넣은 초안 또는 저장 예정 slug(파일명).
- (선택) **`entity/` + `rel/`** 로 나눌지 여부에 대한 힌트.

## 수행 순서

1. 스킬 파일을 읽는다.
2. **`wiki/entity-list.md`**(허브)를 읽어 기존 엔티티·rel·주제문서 인덱스를 파악한다. 입력 키워드와 표의 표시명·원인/결과 요약을 매칭하여 이번 주제와 충돌·중복 가능성을 1~3문장으로 적는다. `entity-list.md`가 없으면 `find`/`Glob`으로 확인한다.
3. 스킬 체크리스트를 복사해 각 항목 아래에 **이번 입력에 대한 구체적 실행 메모**를 한 줄씩 적는다 (체크박스는 `[ ]` 유지).
4. 제안할 `wiki/<slug>.md` 후보와, 넣을 앵커 `id` 후보를 나열한다. **인과 주제**면 추가로: `Entity ID` 후보, 엣지 표 열 설계, `root-cause` / `impact` / `evidence-data` 등 앵커 후보, 연결할 기존 `wiki/*.md`를 적는다.

### 그래프 확장(엔티티·관계) 체크리스트 — 출력에 포함

아래는 **요약·발화·다주체·다중 인과** 입력일 때 `[ ]` 로 그대로 붙여 출력하고, 각 항목 아래에 **이번 입력용 메모**를 한 줄씩 적는다.

```
- [ ] 추출한 Named entity마다 `wiki/entity/<slug>.md` 후보와 H1 한글 표기
- [ ] 인과 쌍마다 `wiki/rel/<cause>-to-<effect>.md` 후보(영문 slug)와 원인/결과 엔티티 매핑
- [ ] 각 `entity` 파일: `## 관련 인과관계`에 걸 `../rel/...` 목록
- [ ] 각 `rel` 파일: 원인·근원 / 결과·영향 / 전달 기제 / 인과 설명 / Evidence(타임코드·수치) 절 설계
- [ ] 각 `rel` 파일: Entity 표·Causality map·엣지 표·`../entity/` 교차링크
- [ ] `wiki/entity-list.md`에 넣을 표 행(표시명·유형·경로)과 rel 인덱스 행 초안
- [ ] `wiki/<주제>.md`가 있을 경우 `entity/`·`rel/`·역링크 연결 계획
- [ ] 상대 경로 검증: 루트↔entity/rel, entity↔rel, entity-list↔entity/rel
```

## 산출물

- 마크다운: 체크리스트 + 메모만. **새 파일 생성이나 기존 파일 편집은 하지 않는다** (사용자가 다음 턴에 `wiki-save-knowledge` 등으로 실행하도록 안내 가능).
